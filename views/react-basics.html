What is React?

React is a JavaSrcipt library created by the Facebook to meet their vision of a single page application. It is designed to create interactive UIs based on encapsulated components that compose the UI. This components allow you to design simple views and set states. React can pass data through your application and update and render the right components when the data changes. Some companies that use React are Facebook, Instagram, Airbnb and Skype..

React uses a "Declarative" language, that means that with it you "say", this should be X or I want this to happen, but you won't say how to do it, you don't give instructions on how something should be done. An example of a declarative language is SQL. In the other hand with an "Imperative" language you describe how you want something to be executed; you specifically give instructions. For example a for loop is Imperative.

Client Side rendering vs. Server Side Rendering

React is for client side rendering. The clear advantage is that your server does less work. It will be use to handle just files, static stuff. In the other hand the browser will do most of the work. This way it's easier to scale your application much further, costing less money. But keep in mind that it doesn't have to be one way or the other, a hybrid approach is best in many cases.

There are different ways to create an application with React. We will be using a syntax called JSX, A compiler called Babel, ES6 and Webpack . Let's begin with JSX.

JSX

JSX is an extension to JavaScript use in React to produce "elements". Do we need to use JSX exclusively? Nope, you can use regular JavaScript but it easier to write it in JSX. JavaScript expressions are units of code that resolve to a value e.g x = 7 or 3 + 2. This expressions are embedded in JSX using curly braces. See below for an example.

const element = <h1>{title}</h1>

But browser don't read JSX they read plain JavaScript. That's what Babel is used for.

Babel

Babel is a compiler to put in JSX or next generation Javascript and get browser compatible JavaScript out. At this point I recommend installing Babel in your text editor. Do that and come back.

ES6 Classes

JavaScript classes provides a much simpler syntax to create objects and deal with inheritance. There are two ways to define a class, as a Class Declaration or as a Class Expression. To access the class, it needs to be defined before it is accessed, this is because they are hoisted.

One more thing. You will notice that each class has a special constructor method. It is use to create and initialize an object within the class. Also right after the constructor the keyword super is used. We use super to call functions on an object's parent. When used with a constructor, super must be used before this can be used. Ok, now that we have explained what a constructor is and the use of super lets take a look at our classes.

How do I create a React application with Node.js

You'll need to install Babel and Webpack as development dependencies.

Webpack

Webpack is a module bundler. It's main purpose is to bundle JavaScript files for usage in a broswer.

Because we use Node.js we need to pass a configuration object as a parameter. You can read more about it here. webpack configutation page

As usual we need to do a few more things to correctly set up our development environment. You'll nee to create a file to write the config code so webpack can access it. You may name this webpack.config.js. Look at the image below for a sample.

npm install -g webpack

To run it:

webpack -w

The flag -w stands for watch. Running webpack like this will continuosly run webpack.

Babel

For webpack and node.js to be able to use babel npm install --save the following packages:

  • babel-core
  • babel-preset-env
  • babel-preset-es2015
  • babel-preset-react
