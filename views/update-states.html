Introduction

In React, any update to the component state will cause the component to be re-rendered with the updated state, so changes may eventually manifest as a UI change as well. But the update can come from different "sources". When the update is in response to event handlers or server responses React offers the setState() method. Notice how in the previous sentence I stated that changes "may eventually" manifest in the UI. This is because setState() does not update the component immediatly. To improve performance, React may update serveral components in a single pass deferring the update until later and therefore not guaranteeing immediate changes. We'll dig into this issue later.

Updating the component local state with setState()
