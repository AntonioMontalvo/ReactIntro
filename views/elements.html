React DOM elements vs. browser DOM elements.

Whilst all HTML elements are element nodes, React DOM elements are objects. They are cheap to create, they are what we want to show on the screen and it is what React renders. The power of React elements is that they live inside a special type of function called Component. In React we have a single root DOM node that is use to render all Components "with their elements". We do this by passing both root and Components to ReactDOM.render().

react and react-dom packages.

To be able to access React in Node.js you are going to add react and react-dom to your package.json file so both dependencies are available in app.js. To install Node.js first go to nodejs.org, click install and follow the instructions. Go to nodejs.org. Then after you are done with npm init, run the code below to install react and react-dom.

npm install react --save npm install react react-dom --save

ReactDOM.render();

This function accepts two parameters. The first one is the Component we want to render, and the second is the single root DOM node in our html where the element or component will be rendered. That is it! The cool thing is that our Component's elements may also have other nested elements and this can have more Components. More on Components later.
