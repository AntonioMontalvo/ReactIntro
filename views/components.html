What is the importance of components in React?

Components are a conerstone in React. The whole idea is to split the UI into independent, reusable pieces allowing an organized arquitecture of components and subcomponents where behavior, logic and layout is self-contained. Components are just functions that accept inputs called props. They return React elements describing what should be render on the screen.

Defining and rendering components.

We are going to use an ES6 class to define components. At its most basic we define the component wich will render a React element and its props. Then adding the user-defined component to an element we can pass JSX attributes to the component as an object that we call props. Because components are just functions, every time we call it we can pass differents props. Let's see this in action demonstrating its reusability concept

Adding State to the component

There is a different between props and state. props are given from a parent to a child or passed to the component; state is purely internal to the component. In other words, when we define a component, it has either a default state or a user-specified state. A property is passed to the component as an attribute or from the parent component to the child component.

In the example above props are given to the component using this.props.type. Now we are going to add Local State to a Class. To do this we add this.state to the constructor right after super(). We assign an object to this local state where all properties internal to the component are initialized with a value.
