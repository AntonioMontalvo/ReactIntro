What exatcly is Lifecycle?

We can declare special methods on the component class to override code at particular times. These methods are called lifecycle hooks and each component can have serveral. With them we can say what to do at some point when something happens.

When an instance of a component is beign created and inserted into the DOM, we call this part of the lifecycle the Mounting phase. Here we have a list of the hooks available for this phase:

  • constructor()
  • componentWillMount()
  • render()
  • componentDidMount()

Another phase of the lyfecycle happens when the components are updated. In the Updating phase, changes to props or state happen. This changes we'll force the component to re-render. Here we have a list of the hooks available for this phase:

  • componentWillReceiveProps()
  • shouldComponentUpdate()
  • componentWillUpdate()
  • render()
  • componentDidUpdate()

The final phase of a component happens when is removed from the DOM. We call this Unmounting. Only one hook here.

  • componentWillUnmount()
